# 部署指南：如何将 Pine Script 学习系统发布到互联网

这个学习系统是一个纯静态网页应用（HTML + CSS + JavaScript），这意味着你不需要购买昂贵的服务器（如阿里云、AWS EC2），只需要使用免费的“静态网站托管服务”即可。

以下是两种最推荐的免费方案：

---

## 方案一：GitHub Pages (推荐程序员使用)

如果你熟悉 Git 操作，这是最稳定的方案。

1.  **创建仓库**：
    *   登录 [GitHub](https://github.com/)。
    *   创建一个新仓库，例如命名为 `pinescript-study`。

2.  **上传代码**：
    *   将 `e:\Quant\PineScript_study\web` 目录下的所有文件上传到该仓库。
    *   你可以通过 Git 命令行上传，或者直接在网页端点击 "Upload files" 将 `web` 文件夹内的内容拖进去。
    *   **注意**：确保 `index.html` 在仓库的根目录下（或者你单独建一个 `docs` 文件夹放进去）。

3.  **开启 Pages 服务**：
    *   进入仓库的 **Settings** (设置) -> **Pages** (侧边栏)。
    *   在 **Build and deployment** 下的 **Source** 选择 `Deploy from a branch`。
    *   在 **Branch** 选择 `main` (或 `master`) 分支，文件夹选择 `/ (root)`。
    *   点击 **Save**。

4.  **访问**：
    *   等待几分钟，GitHub 会给出一个链接，通常是 `https://你的用户名.github.io/pinescript-study/`。

---

## 方案二：Vercel (最简单，速度快)

Vercel 是目前最流行的前端托管平台，支持拖拽部署，无需懂代码。

1.  **注册账号**：
    *   访问 [Vercel.com](https://vercel.com/) 并注册（可用 GitHub 账号登录）。

2.  **安装命令行工具 (可选)**：
    *   如果你安装了 Node.js，可以运行 `npm i -g vercel`。
    *   然后在 `web` 目录下运行 `vercel` 命令即可一键发布。

3.  **网页端部署 (推荐)**：
    *   将 `e:\Quant\PineScript_study` 整个文件夹推送到 GitHub。
    *   在 Vercel 控制台点击 **"Add New..."** -> **"Project"**。
    *   导入你的 GitHub 仓库。
    *   **关键步骤**：在 "Framework Preset" 中保持默认，但在 **"Root Directory"** (根目录) 选项中，点击 Edit 并选择 `web` 文件夹。
    *   点击 **Deploy**。

4.  **访问**：
    *   Vercel 会自动分配一个域名，例如 `pinescript-study.vercel.app`。

5.  **自动更新 (CI/CD)**：
    *   **是的，会自动更新**。这是 Vercel 最强大的功能之一。
    *   当你修改了本地代码（如添加新课程、修改样式）并 `git push` 到 GitHub 仓库后，Vercel 会立刻检测到变更。
    *   它会自动拉取最新代码 -> 重新构建 -> 重新发布。
    *   整个过程通常只需要几十秒。你刷新网页就能看到最新内容。

---

## 方案三：Netlify (拖拽部署)

如果你不想用 Git，这个方法最快。

1.  访问 [Netlify Drop](https://app.netlify.com/drop)。
2.  打开你的电脑文件夹，找到 `e:\Quant\PineScript_study\web`。
3.  将整个 `web` 文件夹直接**拖拽**到 Netlify 网页的虚线框内。
4.  等待几秒钟，部署完成，获得一个随机域名（可以在设置里修改）。

---

## 注意事项

1.  **数据持久化**：
    *   目前的学习进度是保存在用户浏览器的 `localStorage` 中的。这意味着用户在 A 电脑上学的进度，不会同步到 B 电脑或手机上。
    *   如果需要跨设备同步，需要开发后端数据库功能（这会增加部署复杂度和成本）。

2.  **手机访问**：
    *   发布后，直接把生成的网址（URL）发给手机微信，或者生成二维码即可访问。不需要再保证同一 Wi-Fi。

3.  **更新内容**：
    *   如果你修改了 `lessons.json` 或代码，只需要重新上传/推送代码，GitHub Pages 或 Vercel 会自动更新网站。

## 常见问题 (FAQ)

**Q: Vercel 的网页有时间限制吗？会过期吗？**
*   **不会过期**：只要你不主动删除项目，网站会**永久在线**。
*   **没有运行时限**：因为这是一个静态网站（HTML/CSS/JS），它不是按小时计费的服务器，所以没有“关机”或“休眠”的概念。随时访问，随时都在。

**Q: 免费版有什么限制？**
*   **流量限制**：每月 **100GB** 流量。对于个人学习系统来说，这几乎是用不完的（假设你每天访问 100 次，可能连 1GB 都用不到）。
*   **商业限制**：免费版（Hobby Plan）仅限**个人非商业使用**。如果你要在上面卖课或者是公司项目，需要升级。
*   **构建次数**：每天最多部署 6000 分钟（完全够用）。

**Q: 我可以做“付费解锁内容”吗？**
*   **Vercel 免费版**：**不可以**。如果你的网站涉及直接盈利（如售卖课程、付费解锁），根据 Vercel 使用条款，你必须升级到 Pro 方案（约 $20/月）。
*   **GitHub Pages**：**可以**。GitHub Pages 对托管内容的商业性质限制较少，只要内容合法即可。
*   **技术建议**：目前的“付费解锁”只是前端模拟（不安全，懂代码的人可以绕过）。如果你真的要运营付费课程，建议：
    1.  使用专业平台（如 Teachable, 小鹅通）。
    2.  或者开发真正的后端验证系统（需要服务器和数据库）。
